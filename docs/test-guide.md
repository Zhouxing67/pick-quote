# lime 测试指南

> 本文档帮助测试人员快速验证 lime 浏览器扩展的功能正确性。
> 无需编程知识，按步骤操作即可。

---

## 一、环境准备

### 1.1 加载扩展

1. 打开 Chrome，地址栏输入 `chrome://extensions`
2. 右上角开启「开发者模式」
3. 点击「加载已解压的扩展程序」
4. 选择文件夹：`/mnt/d/WSL Build/chrome-mv3-prod`
5. 扩展列表中出现「lime」即加载成功

### 1.2 打开管理页面

- 点击工具栏的 lime 图标，或右键选择「选项」
- 页面应显示：左侧栏（项目管理）+ 右侧主内容区

### 1.3 打开调试控制台（重要）

测试前务必打开控制台，方便发现问题：

- **管理页面**：在管理页面按 `F12` 或右键「检查」→ Console 标签
- **后台脚本**：`chrome://extensions` → lime 的「Service Worker」链接 → Console 标签
- **网页内容脚本**：在任意网页按 `F12` → Console 标签，筛选 `lime` 关键词

> 如果控制台出现红色错误信息，说明有 bug，请截图并记录。

## 三、已知行为与边界情况

以下行为是**设计预期**，不是 bug：

| 行为 | 说明 |
|---|---|
| 复习模式中保存新卡片，当前复习会话不会立即更新 | 复习是封闭会话，新卡片在下次进入复习时出现 |
| 同一内容在同一项目中被去重 | 不同项目可以有相同内容的卡片（项目隔离） |
| 移动卡片到目标项目时，如目标已存在相同内容，会跳过并提示 | 去重机制保护 |
| 关闭管理页面后 badge 仍显示 | badge 由浏览器持久化，下次打开管理页面时会刷新为最新值 |
| 右键菜单在 Service Worker 休眠后首次点击可能延迟 | SW 需要唤醒，属正常 MV3 行为 |
| 删除项目后右键菜单可能短暂残留 | 菜单重建有延迟，刷新页面或等待几秒即可 |

## 四、如何报告问题

### 4.1 基本要求

报告问题时，请提供以下信息：

1. **功能模块**：右键保存 / 项目管理 / 卡片管理 / 复习 / 备份同步 / 设置
2. **操作步骤**：你做了什么（按顺序列出每一步）
3. **预期结果**：你认为应该发生什么
4. **实际结果**：实际发生了什么
5. **截图/录屏**：如果可能，截图或录屏记录问题

### 4.2 如何准确描述问题

#### ❌ 不好的描述

> "保存功能坏了"

#### ✅ 好的描述

> **功能模块**：右键保存
> **操作步骤**：
> 1. 打开 https://example.com
> 2. 选中「Hello World」文字
> 3. 右键 → lime → 加入已有项目 → 选择「测试项目」
> **预期结果**：页面右上角出现「已保存文本到 测试项目」
> **实际结果**：没有任何提示，管理页面中也没有新卡片

### 4.3 如何获取调试信息

如果问题与数据/同步相关，请提供控制台日志：

1. **管理页面控制台**：在管理页面按 `F12` → Console 标签 → 截图红色错误信息
2. **后台脚本控制台**：`chrome://extensions` → lime → 「Service Worker」链接 → 截图 Console
3. **网页控制台**：在出问题的网页按 `F12` → Console 标签 → 搜索 `lime` → 截图相关日志

> 日志中以 `[lime:save]` 开头的行表示保存操作记录
> 日志中以 `[lime:capture]` 开头的行表示网页捕获操作记录

### 4.4 优先级参考

| 严重程度 | 描述 | 示例 |
|---|---|---|
| 🔴 严重 | 数据丢失、功能完全不可用 | 保存卡片后数据消失、扩展无法加载 |
| 🟡 一般 | 功能异常但可绕过 | toast 不显示但卡片实际已保存 |
| 🟢 轻微 | 体验问题、样式异常 | 按钮位置偏移、文字截断 |
